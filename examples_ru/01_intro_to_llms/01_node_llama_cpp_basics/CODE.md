# Разбор кода: основы node-llama-cpp

Этот документ подробно разбирает базовое использование node-llama-cpp, объясняя каждый шаг инициализации и взаимодействия с локальной языковой моделью.

## Обзор

Этот пример демонстрирует базовый рабочий процесс для запуска большой языковой модели (LLM) локально с помощью node-llama-cpp. Он охватывает инициализацию, загрузку модели, создание контекста, а также стандартную и потоковую генерацию текста.

## Импорты и настройка

```javascript
import { getLlama, LlamaChatSession } from 'node-llama-cpp';
import path from 'path';
import {fileURLToPath} from "url";
```

- **`getLlama`**: фабричная функция, возвращающая экземпляр Llama для управления моделями
- **`LlamaChatSession`**: класс для управления диалоговыми взаимодействиями с моделью
- **`path` и `fileURLToPath`**: стандартные утилиты Node.js для работы с путями файлов в ES-модулях

## Шаг 1: Получение экземпляра Llama

```javascript
const llama = await getLlama();
```

**Что делает:** инициализирует базовый экземпляр Llama, который управляет вспомогательными бинарными файлами llama.cpp.

**Зачем это нужно:** этот экземпляр необходим для загрузки моделей и управления низкоуровневыми операциями. Он обеспечивает мост между JavaScript и нативной реализацией llama.cpp.

**Примечание:** это асинхронная операция, так как может потребоваться инициализация нативных бинарных файлов.

## Шаг 2: Загрузка модели

```javascript
const __dirname = path.dirname(fileURLToPath(import.meta.url));
const modelPath = path.join(__dirname, "..", "..", "..", "models", "hf_Qwen_Qwen3-1.7B.Q8_0.gguf");

const model = await llama.loadModel({
    modelPath,
    // gpuLayers: 35, // Uncomment to use GPU acceleration
});
```

**Что делает:** загружает файл модели в формате GGUF в память.

**Ключевые детали:**
- **Путь к модели:** указывает на модель Qwen 3 1.7B, квантизованную до Q8_0 (8-битная квантизация)
- **Формат GGUF:** стандартный формат, используемый llama.cpp для хранения весов модели
- **GPU-слои:** необязательный параметр для переноса слоёв модели на GPU для более быстрого инференса (inference)

**Соображения по производительности:** загрузка модели может занять несколько секунд в зависимости от размера. Модель с 1.7B параметров относительно небольшая и загружается быстро.

## Шаг 3: Создание контекста

```javascript
const context = await model.createContext({
    contextSize: 2048,  // How many tokens to remember
    batchSize: 512,     // Batch size for processing
    threads: 4,         // CPU threads to use
});
```

**Что делает:** создаёт контекстное окно (context window) для работы модели.

**Параметры:**
- **`contextSize: 2048`**: максимальное количество токенов (token), которые модель может запомнить одновременно. Это «рабочая память» модели. Большие значения позволяют вести более длинные разговоры, но потребляют больше оперативной памяти.
- **`batchSize: 512`**: количество токенов для параллельной обработки. Большие батчи (batch) работают быстрее, но потребляют больше памяти.
- **`threads: 4`**: количество потоков CPU для инференса. Обычно больше потоков означает более быструю обработку (в пределах возможностей вашего CPU).

**Использование памяти:** размер контекста напрямую влияет на потребление памяти. Контекст в 2048 токенов является умеренным и подходит для большинства разговоров.

## Шаг 4: Создание чат-сессии

```javascript
const session = new LlamaChatSession({
    contextSequence: context.getSequence()
});
```

**Что делает:** создаёт сохраняющую состояние чат-сессию, которая поддерживает историю разговора.

**Зачем использовать сессию:**
- автоматически управляет историей разговора
- отслеживает использование контекста
- обрабатывает обмен сообщениями в многоходовых разговорах
- упрощает API по сравнению с сырым инференсом модели

**Последовательность:** `context.getSequence()` получает объект последовательности, который управляет состоянием разговора в контекстном окне.

## Шаг 5: Стандартная генерация текста

```javascript
const response = await session.prompt(
    "Explain what a Large Language Model is in one sentence.",
    {
        temperature: 0.7,
    }
);

console.log('Response:', response);
```

**Что делает:** отправляет промпт (prompt) в модель и ждёт полного ответа.

**Параметры:**
- **Первый аргумент:** строка промпта
- **`temperature: 0.7`**: управляет случайностью генерации
    - Низкие значения (0.1–0.5): более детерминированные, сфокусированные ответы
    - Средние значения (0.6–0.8): баланс креативности и связности
    - Высокие значения (0.9–1.2): более креативные, но потенциально менее связные

**Возвращаемое значение:** строка, содержащая полный ответ модели.

**Применение:** лучше всего подходит, когда вы хотите получить полный ответ сразу, например, для вопросов и ответов или когда время ответа не критично.

## Шаг 6: Потоковая генерация текста

```javascript
const tokens = [];
await session.prompt(
    "Count from 1 to 5.",
    {
        maxTokens: 50,
        onResponseChunk: (token) => {
            tokens.push(token.text);
            process.stdout.write(token.text);
        }
    }
);
```

**Что делает:** генерирует текст по токенам по мере его создания моделью.

**Параметры:**
- **`maxTokens: 50`**: максимальное количество токенов для генерации. Предотвращает бесконечную генерацию.
- **`onResponseChunk`**: функция обратного вызова, получающая каждый токен по мере его генерации
    - **`token.text`**: фактический текст токена (может быть словом, частью слова или знаком препинания)

**Почему потоковая передача важна:**
- обеспечивает мгновенную обратную связь для пользователей (как эффект печати в ChatGPT)
- позволяет обрабатывать токены по мере их поступления
- лучший пользовательский опыт для длинных ответов
- позволяет отменить генерацию в процессе при необходимости

**Токен (token) vs. слово:** токен — это не всегда целое слово. Это может быть:
- целое слово: «hello»
- часть слова: «un» + «usual»
- знак препинания: «!»
- пробел

## Шаг 7: Освобождение ресурсов

```javascript
await context.dispose();
```

**Что делает:** освобождает память и ресурсы, занятые контекстом.

**Почему это важно:**
- контексты LLM могут потреблять значительный объём памяти (несколько ГБ для больших моделей)
- предотвращает утечки памяти в долгоживущих приложениях
- хорошая практика, хотя Node.js имеет сборку мусора

**Что очищается:**
- память контекста
- состояние модели
- связанные буферы

## Резюме ключевых концепций

### Трёхуровневая архитектура

1. **Экземпляр Llama** → управляет бинарными файлами и загрузкой моделей
2. **Модель + контекст** → загруженные веса и рабочая память
3. **Чат-сессия** → диалоговый интерфейс с историей

### Иерархия памяти

```
Экземпляр Llama (минимальная память)
    ↓
Модель (несколько ГБ — веса)
    ↓
Контекст (настраиваемый — рабочая память)
    ↓
Сессия (отслеживает состояние разговора)
```

### Асинхронные операции

Каждая основная операция является асинхронной, потому что:
- загрузка модели читает большие файлы с диска
- инференс включает вычислительно intensive операции
- создание контекста выделяет значительный объём памяти

## Типичные паттерны

### Базовые вопросы и ответы
```javascript
const answer = await session.prompt("Your question here");
```

### Потоковый ответ
```javascript
await session.prompt("Your question", {
    onResponseChunk: (token) => console.log(token.text)
});
```

### Управляемая генерация
```javascript
const response = await session.prompt("Question", {
    temperature: 0.3,      // Более сфокусированная
    maxTokens: 100,        // Ограничение длины
    topP: 0.9,             // Nucleus sampling
});
```

## Советы по производительности

1. **Ускорение на GPU:** раскомментируйте `gpuLayers` и установите количество слоёв, которое может обработать ваш GPU
2. **Размер контекста:** используйте минимальный контекст, подходящий для вашей задачи
3. **Размер батча:** большие батчи работают быстрее, но потребляют больше памяти
4. **Потоки:** установите количество ядер вашего CPU (или немного меньше)
5. **Размер модели:** меньшие квантизованные модели (Q4, Q5) работают быстрее, но немного менее точно

## Обработка ошибок

Этот пример не включает обработку ошибок, но в продакшене следует оборачивать операции в блоки try-catch:

```javascript
try {
    const model = await llama.loadModel({ modelPath });
} catch (error) {
    console.error('Failed to load model:', error);
}
```
