# Концепции: LLM и node-llama-cpp

## Что такое большая языковая модель (LLM)?

Большая языковая модель — это система ИИ, обученная на огромных объёмах текстовых данных, которая способна понимать и генерировать текст, близкий к человеческому. Представьте её как очень утончённую систему автодополнения, которая выучила закономерности человеческого языка, прочитав миллионы документов.

### Как работают LLM (упрощённо)

Представьте, что вы пишете кому-то сообщение, и телефон предлагает следующее слово. LLM работает примерно так же, но гораздо более утончённо:

**Этап обучения (выполняется один раз компаниями-разработчиками ИИ):**
- Модель читает миллионы книг, сайтов и статей
- Она выучивает закономерности вроде: «Небо обычно...» → «голубое»
- Это занимает недели и требует мощных компьютеров

**При использовании (инференс):**
- Вы вводите: «Напиши стихотворение о кошках»
- Модель «думает»: «После слов "о кошках" люди обычно пишут...»
- Она выбирает одно слово за другим, собирая ответ по словам
- Каждый выбор слова основан на том, что модель выучила во время обучения

**Важное ограничение:** У модели есть предел памяти. После нескольких страниц разговора она начинает забывать начало (как если бы вы пытались запомнить длинный номер телефона).

### Ключевые особенности

- **Нет доступа в интернет:** LLM не ищут информацию в вебе; они генерируют текст на основе закономерностей, выученных во время обучения
- **Ограничение по дате:** Они «знают» только информацию из своих обучающих данных (до определённой даты)
- **Вероятностный подход:** Ответы не детерминированы — один и тот же промпт может давать разные результаты
- **Ограниченный контекст:** Модель может обрабатывать только ограниченный объём текста за раз

## Что такое node-llama-cpp?

**node-llama-cpp** — это библиотека для Node.js, которая позволяет запускать большие языковые модели локально на вашем собственном компьютере без доступа в интернет или облачных сервисов.

### Технологический стек

```
Your JavaScript Code
        ↓
node-llama-cpp (JavaScript bindings)
        ↓
llama.cpp (C++ inference engine)
        ↓
Your CPU/GPU (actual computation)
```

### Что такое llama.cpp?

**llama.cpp** — это высокопроизводительная реализация на C++ для эффективного запуска LLM на бытовом оборудовании. Именно этот движок оборачивает node-llama-cpp.

**Ключевые инновации:**
- Запуск моделей в оперативной памяти (без необходимости в мощных GPU)
- Поддержка квантизации (сжатие моделей до меньших размеров)
- Работа на CPU, Apple Silicon и GPU NVIDIA/AMD
- Оптимизация скорости и эффективности

### Что такое GGUF?

**GGUF** (GPT-Generated Unified Format) — формат файлов, который llama.cpp использует для хранения весов моделей. Он разработан так, чтобы быть:
- Эффективным для загрузки
- Удобным для работы
- Расширяемым за счёт метаданных

**Пример именования файла:** `model-name-7B.Q8_0.gguf`
- `7B` = 7 миллиардов параметров (размер модели)
- `Q8_0` = уровень квантизации (8-битная квантизация, метод 0)

## Зачем использовать node-llama-cpp?

### Преимущества

- **Конфиденциальность:** Ваши данные никогда не покидают ваш компьютер
- **Без затрат на API:** Запускайте неограниченное количество инференсов бесплатно (после загрузки моделей)
- **Автономная работа:** Работает без подключения к интернету
- **Полный контроль:** Настраивайте всё — от температуры до размера контекста
- **Быстрая итерация:** Нет ограничений API при разработке

### Компромиссы

- **Аппаратные требования:** Необходимо достаточно оперативной памяти (8 ГБ+ для небольших моделей, 16 ГБ+ для более крупных)
- **Размер моделей:** Модели весят несколько гигабайт для загрузки
- **Скорость:** Медленнее облачных API (но быстрее, чем можно ожидать!)
- **Качество:** Небольшие локальные модели менее мощны, чем передовые модели вроде GPT-4

## Ключевые концепции в node-llama-cpp

### Контекстное окно

«Рабочая память» модели — сколько текста она может обработать за раз.

- **Маленькое (512–1024 токена):** Короткие разговоры, простые задачи
- **Среднее (2048–4096 токенов):** Стандартные разговоры, умеренные документы
- **Большое (8192+ токенов):** Длинные документы, обширная история диалога

**Правило thumb:** ~750 слов = 1000 токенов

### Квантизация

Сжатие весов модели для уменьшения потребления памяти при минимальной потере качества.

- **Q2–Q3:** Сильное сжатие, заметная потеря качества
- **Q4–Q5:** Хороший баланс размера и качества (самый популярный вариант)
- **Q6–Q8:** Более высокое качество, файлы больше
- **F16/F32:** Полная точность, самые большие файлы

### Температура

Управляет случайностью генерации:

- **0,0–0,3:** Детерминированный, сфокусированный, повторяющийся
- **0,7–0,8:** Сбалансированный (значение по умолчанию для большинства задач)
- **1,0–1,5:** Креативный, разнообразный, потенциально бессвязный

### Батчинг

Параллельная обработка нескольких токенов для повышения эффективности. Большие размеры батчей работают быстрее, но потребляют больше памяти.

## Типичные сценарии использования

### Для чего node-llama-cpp подходит отлично

- **Чат-боты:** Персональные ассистенты, обслуживание клиентов
- **Генерация контента:** Написание текстов, мозговой штурм, резюмирование
- **Помощь в программировании:** Генерация кода, объяснение, отладка
- **Обработка документов:** Резюмирование, вопрос-ответ по документам
- **Приложения с повышенными требованиями к конфиденциальности:** Здравоохранение, юриспруденция, персональные данные

### Когда лучше использовать облачные API

- **Передовая производительность:** Когда вам нужно качество уровня GPT-4
- **Масштаб:** Обслуживание тысяч одновременных пользователей
- **Мультимодальность:** Визуальные данные, аудио (хотя локальные модели догоняют)
- **Удобство:** Не нужна настройка, обслуживание или специальное оборудование

## Чек-лист для начала работы

1. **Проверка оборудования:** У вас есть 8 ГБ+ оперативной памяти?
2. **Скачивание модели:** Начните с модели на 1–3 млрд параметров
3. **Установка node-llama-cpp:** `npm install node-llama-cpp`
4. **Запуск базового примера:** Убедитесь, что всё работает
5. **Экспериментируйте:** Пробуйте разные модели, температуры и промпты

## Руководство по выбору модели

### По размеру (количеству параметров)

- **1B–3B:** Быстрые, запускаются на любом современном компьютере, подходят для простых задач
- **7B–8B:** Оптимальный вариант для большинства приложений, требуют 8–16 ГБ памяти
- **13B–15B:** Более высокое качество, требуют 16–32 ГБ памяти
- **30B+:** Лучшее качество, требуют 32 ГБ+ памяти или GPU

### По квантизации

Начните с **Q4_K_M** или **Q5_K_M** — они обеспечивают лучший баланс качества и размера для большинства пользователей.

## Общая картина

Локальные LLM совершают революцию в области ИИ, делая его доступным, приватным и бесплатным для запуска. Хотя они и не могут сравниться с крупнейшими облачными моделями по абсолютной производительности, они идеально подходят для множества реальных приложений и с каждым месяцем становятся лучше. node-llama-cpp делает эту технологию доступной для разработчиков JavaScript благодаря простому и знакомому API.
