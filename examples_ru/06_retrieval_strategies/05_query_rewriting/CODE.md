## Обзор

**Файлы:**
- `config.js` — ограничения, карта аббревиатур, фразы для удаления, переключатель LLM.
- `query-rewriter.js` — `normalize`, `classifyIntent`, `stripFiller`, `expandAcronyms`, `stripInjection`, `rewriteHeuristic`, `rewrite`, `parseLLMRewriteResponse`.
- `example.js` — демо: нормализация, удаление заполнителей, расширение аббревиатур, удаление инъекций, полная эвристика, сравнение альтернатив, фильтры для устранения неполадок, асинхронный `rewrite()`, опциональный LLM.

**Структура вывода:**
```javascript
{
  rewrittenQuery: string,
  alternateQueries: string[],
  filters: Record<string, unknown>,
  diagnostics: { intent, removed, stripped, rationale?, source: 'llm'|'heuristic' }
}
```

## Конфигурация

`config.js` предоставляет:
- `maxRewriteLength` — ограничение длины перезаписанного запроса (по умолчанию 300).
- `maxAlternates` — максимальное количество альтернативных запросов (по умолчанию 4).
- `allowedFilterKeys` — допустимые ключи в `filters` (например, category, topic).
- `useLLM` — использовать ли LLM-рефразировку (по умолчанию false).
- `llmTimeoutMs` — тайм-аут для вызова LLM (0 = без тайм-аута).
- `stripPhrases` — фразы-заполнители для удаления (please, can you, thanks и т.д.).
- `acronymMap` — карта сокращений → полных форм (ml → machine learning, k8s → kubernetes).

Переопределение через переменные окружения: `QUERY_REWRITE_MAX_LENGTH`, `QUERY_REWRITE_MAX_ALTERNATES`, `QUERY_REWRITE_USE_LLM`, `QUERY_REWRITE_LLM_TIMEOUT_MS`, `LOG_LEVEL`.

## Нормализация и классификация

**normalize(query)**  
Удаляет пробелы в начале/конце, сворачивает множественные пробелы, удаляет символы нулевой ширины и BOM. Используйте для каждого необработанного ввода.

**classifyIntent(query)**  
Эвристическая классификация для стратегии ретривера:
- `comparison` — «vs», «versus», «difference between», «compare».
- `howto` — «how do», «how to», «steps», «tutorial», «guide».
- `troubleshooting` — «error», «not working», «fix», «debug», «issue».
- `find_doc` — «find», «search», «where is», «locate».
- `factual` — по умолчанию.

Используется для определения альтернатив (например, comparison → генерирует сторона A, сторона B, «A and B comparison») и опциональных фильтров.

---

## Удаление заполнителей и инъекций

**stripFiller(query, stripPhrases)**  
Удаляет фразы вроде «please», «can you», «thanks» (по границам слов, без учёта регистра). Возвращает `{ cleaned, removed }`.

**stripInjection(query)**  
Удаляет инструктоподобный контент, чтобы он не влиял на ретривер:
- «Ignore previous instructions», «You are now…», «System:», «[INST]» и т.д.
Возвращает `{ cleaned, stripped }` (stripped = список описаний совпавших паттернов для логирования).

Используйте удаление (strip) *перед* расширением аббревиатур, чтобы итоговый запрос был чистым и сфокусированным.

---

## Расширение аббревиатур

**expandAcronyms(query, acronymMap)**  
Заменяет токены, совпадающие с ключами карты (по границам слов), на их значения. Сохраняет значение для ретривера (например, «ml» → «machine learning») и предотвращает разбавление эмбеддинга аббревиатурами, которых может не быть в документах.

Конфигурация предоставляет карту аббревиатур по умолчанию; вы можете переопределить или расширить её для конкретной области.

## Эвристическая рефразировка

**rewriteHeuristic(userQuery, config)**  
Запускает полный эвристический конвейер:
1. Нормализация.
2. Классификация намерения (intent).
3. Удаление заполнителей → обновление запроса.
4. Удаление инъекций → обновление запроса.
5. Расширение аббревиатур.
6. Ограничение длины до `maxRewriteLength`.
7. Если намерение — comparison, формирует `alternateQueries` (каждая сторона + «A and B comparison»), ограничивается `maxAlternates`.
8. Опционально устанавливает `filters` (например, troubleshooting → `category: "troubleshooting"`).

Возвращает `{ rewrittenQuery, alternateQueries, filters, diagnostics }`. Синхронный; без LLM.

## LLM-рефразировка и откат

**rewrite(userQuery, config, options)**  
- `options.generate(prompt)` — опционально. Если передано и `config.useLLM` равно true, вызывает LLM со строгим промптом, запрашивающим JSON: `rewrittenQuery`, `alternateQueries`, `filters`, `rationale`.
- `options.conversation` — опционально. Массив `{ role, content }`; добавляется к промпту, чтобы модель могла разрешить «it»/«that».

**parseLLMRewriteResponse(raw)**  
Парсит вывод модели для получения одного JSON-объекта; извлекает и валидирует `rewrittenQuery`, `alternateQueries`, `filters`, `rationale`. Возвращает `null` при ошибке парсинга.

**Поток:**
1. Если `useLLM` и `generate` передано, вызывает `generate(buildRewritePrompt(...))`.
2. Парсит с помощью `parseLLMRewriteResponse`; если валидно и `rewrittenQuery` непустой, применяет ограничения длины/альтернатив и возвращает с `diagnostics.source = 'llm'`.
3. При любой ошибке или невалидном выводе возвращает `rewriteHeuristic(userQuery, config)` с `diagnostics.source = 'heuristic'`.

Таким образом, ретривер всегда получает хотя бы эвристическую рефразировку; LLM — опциональное улучшение.

## Запуск примеров

Из корня репозитория:
```bash
node examples/06_retrieval_strategies/05_query_rewriting/example.js
```

Для примеров 1–8 модель не требуется (только эвристика). Пример 9 запускает LLM-рефразировку, если `QUERY_REWRITE_USE_LLM=1` и доступен путь к модели LLM (например, `LLM_MODEL_PATH` или путь к Qwen по умолчанию); в противном случае выводит короткое сообщение и показывает эвристический откат.

## Лучшие практики

- Всегда нормализуйте и запускайте удаление инъекций, чтобы ретривер никогда не видел инструкций.
- Держите расширение аббревиатур специфичным для области и настраиваемым.
- Используйте один вызов LLM со строгим JSON-контрактом и валидатором; не зацикливайте и не используйте инструменты.
- Логируйте диагностику (intent, removed, stripped, source) для отладки и настройки.
- Ограничивайте длину перезаписанного запроса и количество альтернатив, чтобы избежать бесконтрольных затрат и задержек.
