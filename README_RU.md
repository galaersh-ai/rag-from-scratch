# RAG от нуля

**Разберите Retrieval-Augmented Generation (RAG), создав его самостоятельно — шаг за шагом.**
Без чёрных ящиков. Без облачных API. Только понятные объяснения, простые примеры и локальный код, который вы полностью понимаете.

Этот проект следует той же философии, что и [AI-агенты от нуля](https://github.com/pguso/ai-agents-from-scratch):
сделать продвинутые концепции AI доступными для разработчиков через минимальный, хорошо объяснённый, реальный код.

---

## Что вы узнаете

- **Что такое RAG на самом деле** и почему он так эффективен для поиска знаний.
- **Как работают эмбеддинги (embedding)** — превращение текста в числа, понятные модели.
- **Как создать локальную векторную базу данных** — хранение и поиск документов с высокой производительностью.
- **Как соединить всё вместе** — извлечение контекста и подача его в LLM для обоснованных ответов.
- **Как делать переупорядочивание (reranking) и нормализацию** — повышение точности ретривера и снижение шума.
- **Предобработка запросов (query rewriting)** — нормализация и перезапрос пользовательских запросов (эвристические и LLM) перед ретривером.
- **Пошаговый разбор кода** — каждая функция объяснена, ничего не скрыто.

---

## Концептуальный обзор

Retrieval-Augmented Generation (RAG) усиливает языковые модели, давая им доступ к **внешним знаниям**.
Вместо того чтобы заставлять модель «помнить» всё, вы даёте ей возможность **извлекать релевантный контекст** перед генерацией ответа.

**Конвейер:**
1. **Требования к знаниям** — определение вопросов и потребностей в данных.
2. **Загрузка данных** — импорт и структурирование документов.
3. **Разбиение текста и нарезка (chunking)** — разделение данных на управляемые фрагменты.
4. **Эмбеддинг** — превращение фрагментов в числовые векторы.
5. **Векторное хранилище (vector store)** — сохранение и индексация эмбеддингов для быстрого поиска.
6. **Ретривер (retrieval)** — извлечение наиболее релевантного контекста для заданного запроса.
7. **Переупорядочивание после ретривера (post-retrieval re-ranking)** — переранжирование результатов для приоритизации лучшего контекста.
8. **Предобработка запросов и нормализация эмбеддингов** — очистка и стандартизация входных векторов для согласованности.
9. **Аугментация** — объединение извлечённого контекста с промптом модели.
10. **Генерация** — создание обоснованных ответов с помощью локальной LLM.

**Библиотека:** загрузчики, сплиттеры текста, эмбеддинги, векторные хранилища, ретриверы, цепочки, промпты (см. «Структура проекта»).

### Что дальше

Следующие темы будут добавлены **пошагово** в ближайшие **недели и месяцы**:

**Стратегии ретривера:**
- Ранжирование и подсчёт оценок результатов

**Инжиниринг промптов для RAG:**
- Техники наполнения контекста (context stuffing)

- Сжатие контекста

**RAG в продакшене:**
- Обработка ошибок и откаты (fallback)
- Потоковые ответы
- Примеры полного конвейера RAG

**Оптимизация и оценка:**
- Метрики ретривера (точность, полнота, MRR)
- Метрики качества генерации
- Фреймворки end-to-end оценки

**Продвинутые функции:**
- Наблюдаемость (observability) и мониторинг производительности
- Стратегии кэширования для повторяющихся запросов
- Обработка метаданных и структурированных данных
- Интеграция с графовой базой данных (например, kuzu)
- Мультимодальный RAG

**Шаблоны и руководства:**
- Готовые стартовые шаблоны (простой RAG, API-сервер, чатбот)
